### 1b. Customer card order → authorize
- **Where:** `/order/<slug>` in incognito.
- [ ] Add an item → checkout → choose **pay by card** → pay with `4242…`.
- [ ] You land on the **confirmation** page.
- [ ] In the kitchen, the order appears as a **new/pending** card order (payment status **authorized**).
- **Fail if:** the card option is missing, payment errors, or the order never reaches the kitchen.

### 1c. Accept → capture (money moves)

- [ ] Payment status flips to **paid**.

### 1d. Reject pre-accept → void (no charge)
- [ ] Place another card test order; in the kitchen **Reject** it before accepting.
- [ ] Stripe shows the authorization **canceled/voided** — customer never charged.

## 2. ⏱️ Kitchen 4-minute alert + full GloriaFood audio
- **Where:** `/kitchen` (tablet), hard-refreshed, sound enabled, alert sound = **GloriaFood**.

- [ ] The **full ~4-minute GloriaFood audio** plays through (don't accept) — it should NOT cut off around 3:00.

## 3. ↩️ Refund (partial + full) from the kitchen
- **Prereq:** a **paid** card order (from 1c).
- **Where:** `/kitchen` → open the order → **Refund** button (only shows on paid card orders).
- [ ] **Partial:** choose Partial, enter an amount less than the total → **Issue refund** → success; badge shows **PARTIALLY REFUNDED**; "refunded so far" line appears; Stripe shows the partial refund.
- [ ] **Full / remainder:** open Refund again → Full → issue → badge **REFUNDED**; Stripe shows the order fully refunded.
- [ ] Try to refund **more than remaining** → blocked with an error.
- **Fail if:** refund button missing on a paid card order, refund exceeds total, or Stripe doesn't reflect it.

## 4. 🍽️ Menu item day/time availability
- **Where:** `/admin/menu` → edit an item → set **Available days** (e.g. weekdays only) + **Available from/to** (e.g. a window that is **active right now**).
- [ ] Back in the menu list, the item shows a **clock badge** (the time window, or "Selected days").
- [ ] On `/order/<slug>` during the window → item is **orderable**.
- [ ] Change the window to one that is **NOT active now** (e.g. 2 hours in the past) → on `/order/<slug>` the item is **hidden**.
- [ ] (Server guard) With the item out-of-window, it cannot be ordered even via a stale tab — placing it errors.
- **Fail if:** no badge, or an out-of-window item can still be ordered.

> Note: day/time is evaluated in the **restaurant's timezone**.
